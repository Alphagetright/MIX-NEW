一、引言
================================================================================

1.1 项目背景

唐诗是中国古典文学的瑰宝，其意象运用的精妙程度在世界诗歌史上独树一帜。
从李白的"床前明月光"到杜甫的"感时花溅泪"，从王维的"大漠孤烟直"到
李商隐的"春蚕到死丝方尽"，唐诗中的意象承载着丰富的文化内涵和情感表达。

传统文学研究中，意象标注工作主要依赖研究者的手工操作，存在以下突出问题：
第一，标注工作量大，处理数百首诗歌需要耗费大量人力和时间；第二，主观性强，
不同研究者对同一意象的分类和解读可能存在差异；第三，难以进行跨文本的
系统性比较和统计分析；第四，研究成果难以数字化保存和共享。

为解决上述问题，本系统以认知诗学理论为指导，综合运用自然语言处理技术、
向量检索（RAG）和大语言模型生成能力，实现对唐诗意象的系统化提取、
分类标注、情感分析和多维度统计。系统旨在为古典文学研究者、教育工作者
和诗歌爱好者提供一个集数据管理、智能检索、认知分析和可视化展示于一体的
综合研究平台。

1.2 软件概述

唐诗意象智能分析系统是一款基于 B/S 架构的 Web 应用程序，采用前后端分离
的设计模式。后端使用 Python Flask 框架提供 RESTful API 和 SSE 流式接口，
前端采用原生 HTML/CSS/JavaScript 技术栈配合 ECharts 图表库实现数据可视化，
数据持久化层使用 ChromaDB 向量数据库实现语义检索功能，大语言模型调用层
兼容 OpenAI API 格式的本地或远程推理服务。

系统在架构设计上采用模块化分层思想，主要分为以下层次：
- 配置层（config.py）：集中管理所有系统配置参数
- 日志层（logger.py）：提供分级日志记录和文件滚动功能
- 缓存层（cache.py）：实现内存缓存和文件缓存双重机制
- 校验层（errors.py、validators.py）：统一异常处理和输入校验
- 服务层（analytics.py、export_service.py、preprocessor.py 等）：核心业务逻辑
- 数据模型层（models.py）：定义诗歌、意象、分析单元等数据结构
- 中间件层（middleware.py）：请求追踪、CORS、性能监控
- 展示层（templates/index.html、admin.html）：前端用户界面

系统核心功能包括数据图谱可视化展示、基于 RAG 的智能问答、认知诗学意象
深度解析、多维度统计分析、数据导出备份以及系统运维管理等。

1.3 运行环境

本系统的运行环境要求如下：

  操作系统     Windows 10/11、Linux、macOS
  Python版本   Python 3.10 及以上（推荐 3.13）
  Web浏览器    Chrome 90+、Edge 90+、Firefox 90+（推荐 Chrome）
  大模型服务   兼容 OpenAI API 格式的 LLM 推理服务
  磁盘空间     不低于 1 GB（含向量库、日志、导出文件）
  内存         推荐 4 GB 以上（大模型推理需额外显存）
  网络         本地运行时无需互联网连接（大模型服务可在本地或局域网部署）

二、安装与部署
================================================================================

2.1 环境要求

本系统依赖以下 Python 第三方库：

  依赖库          版本要求       用途
  Flask           >= 3.0.0      Web 框架，处理 HTTP 请求与路由
  Requests        >= 2.28.0     HTTP 客户端，调用 Embedding 和 Chat API
  ChromaDB        >= 0.4.0      向量数据库，存储和检索诗歌语义向量
  openpyxl        >= 3.1.0      Excel 文件读写（可选，用于导出功能）
  fpdf2           最新版         PDF 文件生成（可选，用于软著材料生成）
  pytest          >= 8.0.0      单元测试框架（可选，仅开发测试需要）

  【注意】ChromaDB 在首次运行时可能自动下载依赖的 tokenizer 模型文件，
  请确保网络连接正常。如果使用离线环境，可预先下载所需模型文件。

2.2 安装步骤

  步骤一：检查 Python 环境

    打开终端（Windows 下使用 CMD 或 PowerShell），输入以下命令确认
    Python 已正确安装：

        python --version

    如果能显示 Python 3.10.x 或更高版本，则环境准备就绪。

  步骤二：安装项目依赖

    在项目根目录（包含 requirements.txt 的目录）下执行：

        pip install -r requirements.txt

    如果安装速度较慢，可使用国内镜像源加速：

        pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

  步骤三：验证安装（可选）

    运行系统自带的单元测试，确认所有模块正常工作：

        python -m pytest tests/ -v

    如果看到 "143 passed" 则表示所有模块安装正确。

2.3 启动方式

  Web 服务启动：

    在项目根目录下执行：

        python app.py

    启动成功后终端显示类似以下信息：

        * Running on http://127.0.0.1:5000
        * Running on http://192.168.1.102:5000

    注意：终端窗口必须保持打开状态，关闭窗口将终止服务。

  Windows 中文乱码解决：

    如果在 Windows 终端下显示乱码，请使用以下命令启动：

        PYTHONIOENCODING=utf-8 python app.py

2.4 访问系统

  打开 Web 浏览器，在地址栏输入：

        http://127.0.0.1:5000

  即可进入系统首页。注意：不要使用 0.0.0.0:5000 访问。

  管理后台地址：

        http://127.0.0.1:5000/admin/

  局域网内其他设备访问：

    在同一局域网内，其他设备可通过服务器的局域网 IP 地址访问，
    例如 http://192.168.1.102:5000。

3.1 数据图谱模块

  数据图谱是系统的主界面（首页），提供意象数据的可视化展示和交互式浏览
  功能。页面顶部为统计概览栏，中部为两张统计图表，下部为筛选器和溯源
  数据表。

  3.1.1 统计概览

    页面顶部的统计条以横向卡片形式展示三项核心指标：
    - 净解析诗歌：系统当前加载的诗歌总数（自动去重后的唯一值）
    - 提取真意象：从诗歌中提取的意象标注条目总数
    - 分类维度：系统支持的意象分类层级（4 层 10 维）

    统计条右侧显示"回收站"按钮，点击可查看和管理已删除的意象条目。

  3.1.2 核心意象 Top50 图表

    意象文本。图表具备以下交互功能：

    - 鼠标悬停：显示意象文本和出现次数
    - 底部滑块：拖拽缩放查看范围，支持从 50 个中选取部分查看
    - 点击柱条：自动将对应的意象文本填入搜索框并筛选数据表
    - 全屏查看：点击卡片右上角"放大视图"按钮可全屏显示

    图表右侧显示"意象分类域分布"柱状图，展示各分类域的意象数量分布。

    【插入图3-2：核心意象 Top50 柱状图展示】

  3.1.3 意象分类域分布图表

    页面右上方展示分类域分布柱状图，将意象按大类归属进行聚合统计。
    分类域包括：
    - 自然意象：天文意象、地理意象、植物意象、动物意象
    - 社会意象：生产生活意象、军事战争意象、制度观念意象

    - 分类域下拉框：按意象大类归属筛选
    - 体裁/标签下拉框：按诗歌体裁或分类标签筛选
    - 情感倾向下拉框：按情感类别筛选

    当有筛选条件激活时，"清除筛选"按钮自动显示，点击可一键重置所有

3.2 AI 智能问答模块

  AI 问答功能基于 RAG（Retrieval-Augmented Generation，检索增强生成）
  架构，将向量检索与大语言模型生成能力相结合，实现基于诗歌原文资料的
  智能问答。

  3.2.1 提问方式

    在输入框中输入关于唐诗的自然语言问题。系统会自动分析问题意图，
    识别问题中涉及的诗人（如果有的话），并进行针对性的语义检索。

    建议提问示例：
    - "杜甫写过哪些关于秋天的诗？"
    - "李白的《静夜思》中用了哪些意象？"
    - "唐诗中'月'的意象表达了哪些情感？"
    - "王维的山水诗中常见的意象有哪些？"
    - "白居易的诗有什么特点？"

  3.2.2 检索与回答流程

    点击"检索并生成回答"按钮后，系统执行以下流程：

    1. 意图识别：自动检测问题中是否包含已知诗人姓名。
       如果检测到（如"杜甫"、"李白"），则在向量检索时按诗人过滤。
       同时清理问题中的无关词语（如"写过"、"哪些"等）。

    2. 语义检索：将问题文本通过 Embedding API 转换为向量，
       在 ChromaDB 向量库中按余弦相似度检索最相关的诗歌原文，
       默认返回相似度最高的 5 条结果。

    3. 上下文构建：将检索到的诗歌原文按标准格式组织为上下文，
       包含诗名、作者和完整原文。

    4. 流式生成：将问题和上下文一起发送给大语言模型，并以 SSE
       （Server-Sent Events）流式方式逐字推送回答内容到前端，
       实现实时显示效果。

    5. 结果展示：左侧面板展示检索到的诗歌卡片，右侧展示 AI 回答。

  3.2.3 检索结果展示

    检索到的诗歌以卡片列表形式展示在左侧面板，每张卡片包含：
    - 诗名和作者
    - 相似度评分（0-1 之间的数值，越接近 1 表示越相关）
    - 诗歌原文预览（前 120 字）
    - 点击卡片可展开查看全文
    - "点击卡片展开全文"提示文字

    如果未检索到相关诗歌，系统会显示"暂无检索结果"提示。

    【插入图3-6：AI问答界面，显示输入框、检索结果卡片列表、回答区域】

  3.2.4 多轮追问

    在 AI 回答生成完成后，底部会出现追问输入框。用户可以继续输入问题，
    系统会保留之前的对话历史作为上下文，实现连续的多轮交互。

    追问功能的特点：
    - 自动保存对话历史（user 和 assistant 角色交替）
    - 每次追问携带全量历史记录发送给大模型
    - 支持对前一轮回答的深入追问和细节探讨
    - 新的首轮提问会重置对话历史

    【插入图3-7：AI问答多轮追问界面】



3.3 认知诗学意象解析模块

  本功能面向专业研究场景，对单个意象进行深度认知诗学分析。采用右侧
  抽屉式交互设计，支持完整的解析流程和多轮追问。

  3.3.1 功能入口

    在数据图谱页面的溯源数据表中，每一行意象条目都包含"解析"操作按钮。
    点击任一行的"🔍解析"按钮，即可从浏览器右侧展开解析抽屉。

    抽屉设计特点：
    - 从右侧平滑滑入，带有半透明遮罩层
    - 宽度可拖拽调整（320px 到窗口宽度减 80px）
    - 解析状态自动缓存，切换意象不丢失进度
    - 切换意象时自动恢复对应缓存

  3.3.2 解析界面布局

    解析抽屉从上到下依次分为五个区域：

    1. 头部区域：显示意象文本（蓝色高亮）、所属诗歌名称、结构功能组
       信息和所在诗句（楷体显示）。

    2. 标签区域：以彩色标签形式展示该意象的多维标注信息，包括：
       - 感知通道（如视觉、听觉）
       - 分类（如自然意象-天文意象）
       - 表现功能
       - 认知强度
       - 情感极性 + 情感类别

    3. 对话区域：显示 AI 解析结果和交互历史，支持 Markdown 渲染。
       首次打开时自动发送默认解析指令。

    4. 输入区域：底部固定，支持输入追问文本，按 Enter 或点击发送。

    5. 拖拽手柄：抽屉左侧边缘的拖拽条，可左右拖动调整抽屉宽度。

    【插入图3-8：认知诗学解析抽屉界面】

  3.3.3 五维解析维度

    AI 解析严格基于意象的结构化标注数据，从以下五个维度进行分析：

    （1）感知层
    分析意象触发的感官体验和通感效果。例如，"月"的视觉形象如何
    触发清冷、孤寂的感官体验。

    （2）文化层
    追溯意象的典故来源和文化符号意义。例如，"柳"在唐代诗歌中
    作为送别象征的文化渊源。

    （3）情感层
    基于情感极性和情感类别标注，分析意象在诗歌中的情感功能。
    例如，"落日"意象如何通过"负面"情感极性传达感伤情绪。

    （4）结构层
    分析意象作为特定结构功能组在诗歌中的结构作用，如上句铺垫、
    下句点题等。

    （5）跨诗比较
    比较同一意象在不同诗人、不同诗歌中的使用异同，揭示意象的
    多义性和演变规律。

    系统提示 AI 严格基于提供的数据进行分析，禁止编造不在数据中的内容。

  3.3.4 多轮追问交互

    在首次解析完成后，用户可在底部输入框输入追问内容。系统会将
    历史对话（包含首次的 system prompt 和问答）一并发送给大模型，
    支持以下追问场景：

    - 对某个解析维度的深入探讨（如"请详细分析这个意象的文化层"）
    - 请求补充其他维度（如"从结构层再分析一下"）
    - 跨意象比较（如"这个意象和上一条解析的'月'有什么不同"）

    每次追问后，对话区域自动滚动到底部，显示最新的回答内容。
    关闭抽屉后再次打开同一意象，之前的对话历史自动恢复。

5.1 配置文件结构

  系统所有配置参数集中在项目根目录的 config.py 文件中，采用模块级常量
  定义方式。主要配置分类如下：

  配置分类            内容
  API 配置            大模型服务地址、Embedding 和 Chat 模型名
  路径配置            数据目录、向量库目录、导出目录、日志目录
  RAG 检索配置        检索数量、集合名称、距离计算方式
  Flask 配置          服务端口、调试模式、密钥
  数据分页            每页显示条数、最大渲染条数
  意象分类映射        大类和小类编码到中文名称的映射
  诗人列表            系统内置的已知诗人名单
  日志配置            日志级别、文件大小、备份数量、格式
  缓存配置            TTL 时间、缓存启用开关
  导出配置            CSV 编码格式

5.2 常用配置项

  以下为需要经常修改的配置项及其说明：

  FLASK_PORT = 5000
    说明：Web 服务监听端口。如果 5000 被占用，可改为其他端口（如 8080）。
    修改后需重启服务，访问地址相应改为 http://127.0.0.1:新端口。

  EMBED_API_URL = "http://127.0.0.1:44787/v1/embeddings"
    说明：文本向量化服务的 API 地址。如果使用不同的 Embedding 服务，
    修改此地址和对应的 EMBED_MODEL 名称。

  CHAT_API_URL = "http://127.0.0.1:44787/v1/chat/completions"
    说明：大语言模型 Chat 服务的 API 地址。支持任何兼容 OpenAI 格式的
    服务（包括本地部署的 vLLM、TGI 或云端 API）。

  TOP_K = 5
    说明：向量检索时返回的最相似结果数量。增大此值可提供更多上下文，
    但会增加 Token 消耗；减小此值可提高回答的精准度。

  CACHE_DEFAULT_TTL = 300
    说明：缓存默认生存时间（秒）。300 秒 = 5 分钟。设为 0 表示永不过期，
    设为负数表示禁用缓存。

  LOG_LEVEL = "INFO"
    说明：日志输出级别。可选 DEBUG、INFO、WARNING、ERROR。DEBUG 级别
    会输出更详细的调试信息，适合开发环境；生产环境建议使用 INFO 或
    WARNING。

5.3 数据源配置

  系统的数据文件按用途存放在两个目录中：

  (1) poem_json/ 目录
  用途：系统启动时自动加载，生成溯源数据和统计分析。
  格式：JSON 或 TXT 文件，包含诗歌编号、标题、作者、诗行、分析单元等
        结构化字段。
  文件示例：tang_poetry_313.json（313 首唐诗的完整标注数据）

  (2) json_dataa/ 目录
  用途：RAG 向量库构建脚本（build_rag.py）的输入数据。
  格式：JSON 文件，包含"诗歌集"数组，每首诗歌至少需要诗歌编号、
        标题、作者、原文四个字段。
  注意：该目录与 poem_json/ 可以相同，也可以不同，由具体使用场景决定。

  如需要更新数据，只需将新的 JSON 文件放入对应目录，然后执行以下操作：
  - 刷新前台数据：在管理后台点击"刷新数据缓存"或重启 app.py
  - 更新向量库：运行 python build_rag.py 重新构建

6.1 日志系统

  日志文件位置

    系统日志默认保存到项目根目录的 logs/app.log 文件中。
    首次启动时自动创建 logs/ 目录。

  日志滚动机制

    - 单个日志文件最大 10 MB（约 5 万行日志）
    - 超过上限后自动创建新文件，旧文件重命名为 app.log.1
    - 保留最近 5 个备份文件（app.log.1 ~ app.log.5）
    - 最旧的日志文件自动删除，无需手动清理

  日志级别

    DEBUG    输出所有日志，包括调试信息，适合开发和问题排查
    INFO     输出常规运行信息，如启动、请求、数据加载（默认级别）
    WARNING  仅输出警告和错误信息，适合生产环境减少日志量
    ERROR    仅输出错误信息，仅在出问题时记录

  日志格式

    每行日志包含以下信息：
    时间戳 [级别] 模块名: 消息内容

    示例：
    2026-05-10 14:24:50,052 [INFO] app: 启动唐诗意象智能分析系统
    2026-05-10 14:24:55,231 [INFO] app: 数据加载完成: 313 首诗歌, 2345 条意象

  控制台同步输出

    所有日志信息同时输出到控制台（终端窗口）和日志文件，方便实时监控。

6.2 缓存管理

  系统采用双层缓存策略：

  内存缓存（MemoryCache）
    - 使用线程安全的字典存储（threading.RLock）
    - 读写速度快，适合频繁访问的热数据
    - 默认 TTL：5 分钟（可配置）
    - 程序重启后自动清空
    - 可通过管理后台"清理缓存"按钮或 CLI 命令清空

  文件缓存（FileCache）
    - 持久化到磁盘 cache/ 目录，JSON 格式存储
    - 跨会话保留，程序重启后仍然有效
    - 默认 TTL：5 分钟（可配置）
    - 支持特殊字符键名（自动替换 / \ : 为 _）
    - 可通过管理后台或 CLI 命令清空

  缓存清理方式：

  方式一：管理后台
    访问 http://127.0.0.1:5000/admin/ 点击"清理缓存"按钮

  方式二：CLI 命令
    python cli.py clear-cache

  方式三：API 接口
    POST http://127.0.0.1:5000/admin/api/cache/clear
    Content-Type: application/json
    Body: {"type": "all"}

  缓存装饰器：

    系统提供 @cached 装饰器，可方便地对函数返回值进行缓存：
    @cached(ttl=300)
    def expensive_function():
        # 耗时操作
        return result

6.3 向量库管理

  向量数据库（ChromaDB）保存在项目根目录的 rag_db/ 文件夹中。

  构建向量库：

    运行以下命令从 json_dataa/ 目录读取数据重新构建：
    python build_rag.py

    构建过程：
    1. 删除旧库（如果存在）
    2. 读取目录下所有 JSON 文件
    3. 遍历每首诗歌，提取原文
    4. 调用 Embedding API 生成向量
    5. 将向量和元数据写入 ChromaDB
    6. 每首诗间隔 0.3 秒，避免 API 限流

  检查向量库状态：

    python cli.py check-rag

    输出示例：
      向量库状态:
        路径: /path/to/rag_db
        集合: poems
        向量数: 313
        示例文档:
          [1] 《静夜思》李白 (30字)
          [2] 《春晓》孟浩然 (24字)

  重新构建场景：

    以下情况需要重新构建向量库：
    - json_dataa/ 目录中添加了新的诗歌数据
    - 修改了现有诗歌的原文内容
    - 切换了 Embedding 模型
    - 向量库文件损坏或丢失

6.4 数据备份

  系统内置数据备份功能，可通过以下方式操作：

  自动备份脚本：
    运行 preprocessor.py 中的 backup_data() 函数：
    会在项目根目录创建 backup_poem_json_时间戳/ 文件夹，

七、常见问题
================================================================================

7.1 启动与访问问题

  Q1：启动后浏览器无法访问页面？

  A1：可能的原因和解决方法：
    原因1：终端窗口已关闭导致服务终止。
    解决：重新运行 python app.py 并保持终端窗口打开。
    原因2：浏览器地址输入错误。
    解决：在地址栏输入 http://127.0.0.1:5000（注意不是 0.0.0.0:5000）。
    原因3：端口被其他程序占用。
    解决：修改 config.py 中的 FLASK_PORT 为其他端口（如 8080）。

  Q2：启动时报错 "Address already in use"？

  A2：说明 5000 端口已被占用。解决方法：
    方法1：关闭占用端口的程序
    方法2：修改 config.py 中的 FLASK_PORT 为其他值
    方法3：在终端中运行 netstat -ano | findstr 5000 查看占用进程 PID，
           然后在任务管理器中结束该进程

  Q3：如何在其他电脑上访问本系统？

  A3：在同一局域网内的其他设备上：
    1. 在运行系统的电脑上查看局域网 IP：
       Windows：在 CMD 中运行 ipconfig，找到 IPv4 地址
    2. 其他设备浏览器中输入：http://IP地址:5000
    例如：http://192.168.1.102:5000
    注意：跨网段访问需要配置路由或使用 VPN。

  Q4：控制台输出中文乱码如何解决？

  A4：Windows 终端默认编码为 GBK，与 UTF-8 不兼容。解决方法：
    启动命令前加 PYTHONIOENCODING=utf-8 环境变量：
      PYTHONIOENCODING=utf-8 python app.py
    或者在 CMD 中先执行 chcp 65001 切换到 UTF-8 编码。

7.2 功能使用问题

  Q5：AI 问答没有返回任何结果？

  A5：请按以下顺序排查：
    步骤1：确认已运行 python build_rag.py 构建了向量库
    步骤2：检查 rag_db/ 目录是否存在且不为空
    步骤3：确认大模型 API 服务正在运行：
      - 默认地址：http://127.0.0.1:44787
      - 可以在浏览器中访问 http://127.0.0.1:44787/v1/models 测试
    步骤4：在 config.py 中检查 API 地址和模型名是否正确
    步骤5：查看 logs/app.log 中的错误信息

  Q6：检索到的诗歌与问题不相关？

  A6：可能的原因：
    原因1：向量库中的诗歌数据不包含相关内容
    解决：补充 json_dataa/ 目录中的数据并重新构建向量库
    原因2：TOP_K 值设置过小
    解决：在 config.py 中将 TOP_K 增大（如改为 10）
    原因3：Embedding 模型的语义理解能力有限
    解决：尝试更换更强的 Embedding 模型

  Q7：数据表中没有显示任何数据？

  A7：可能原因：
    原因1：poem_json/ 目录中没有可用的数据文件
    解决：将诗歌标注 JSON 文件放入 poem_json/ 目录
    原因2：JSON 文件格式不正确
    解决：运行 python cli.py scan 检查数据文件解析状态
    原因3：数据缓存为初始状态
    解决：在管理后台点击"刷新数据缓存"

  Q8：意象解析抽屉打开后长时间无响应？

  A8：可能原因：
    原因1：大模型 API 服务未运行或地址错误
    解决：确认 LLM API 服务可用性
    原因2：网络连接超时
    解决：在 config.py 中增大 CHAT_TIMEOUT 值
    原因3：模型负载过高
    解决：等待其他请求完成后重试

7.3 系统配置问题

  Q9：如何修改大模型 API 的地址和模型？

  A9：编辑 config.py 中的以下配置项：
    EMBED_API_URL = "http://新地址:端口/v1/embeddings"
    CHAT_API_URL = "http://新地址:端口/v1/chat/completions"
    EMBED_MODEL = "新模型名"
    CHAT_MODEL = "新模型名"
    修改后保存文件，重启 python app.py 即可生效。

  Q10：如何添加新的诗人到检索过滤列表？

  A10：编辑 config.py 中的 KNOWN_AUTHORS 列表，添加诗人姓名：
    KNOWN_AUTHORS = [
        "杜甫", "李白", "王维", "白居易",
        "新增诗人名",  # 在这里添加
    ]
    添加后 AI 问答功能即可自动识别新诗人的姓名。

  Q11：如何调整数据表的分页大小？

  A11：编辑 config.py 中的以下两个配置项：
    PAGE_SIZE = 25      # 每页显示条数（默认 25）
    RENDER_LIMIT = 5000  # 最大渲染条数（默认 5000）
    增大 RENDER_LIMIT 可显示更多数据，但可能影响页面性能。

  Q12：如何修改日志级别查看更多调试信息？

  A12：编辑 config.py 中的 LOG_LEVEL 配置项：
    LOG_LEVEL = "DEBUG"  # 输出最详细的信息
    修改后重启服务，日志将包含更多调试信息，有助于排查问题。

  C.3 软著申请表填写参考

    面向领域/行业：
    中国古典文学数字化研究领域，面向高校中文系、古典文学研究所、
    图书馆数字人文中心及诗歌教育行业。

    软件的主要功能：
    1. 意象数据管理：批量加载 JSON/TXT 标注数据，支持搜索、筛选、
       分页浏览、删除恢复
    2. 可视化分析：ECharts 图表展示核心意象 Top50 和分类域分布，

    4. 双层缓存策略（内存 + 文件，TTL 支持）减少重复计算与 API 调用
    5. 模块化分层架构：配置、日志、缓存、校验、服务层独立解耦
