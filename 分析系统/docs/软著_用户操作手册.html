一、引言

1.1 项目背景

唐诗是中国古典文学的瑰宝，其意象运用的精妙程度在世界诗歌史上独树一帜。传统文学研究中，意象标注工作量大、主观性强、难以跨文本比较，长期以来缺乏系统化的计算机辅助分析工具。

本系统以认知诗学理论为指导，综合运用自然语言处理、向量检索（RAG）和大语言模型技术，实现对唐诗意象的系统化提取、分类标注、情感分析和多维度统计。旨在为古典文学研究者、教育工作者和诗歌爱好者提供一个集数据管理、智能检索、认知分析和可视化展示于一体的综合研究平台。

1.2 软件概述

唐诗意象智能分析系统是一款 B/S 架构的 Web 应用程序。系统后端使用 Python Flask 框架，前端采用原生 HTML/CSS/JavaScript 配合 ECharts 图表库，数据库层使用 ChromaDB 向量数据库实现语义检索，大语言模型调用层兼容 OpenAI 格式的本地或远程 API。

系统主要包含以下核心功能模块：

  • 数据图谱：意象数据的可视化展示、多维度筛选与溯源浏览
  • AI 智能问答：基于向量检索增强生成（RAG）的诗歌知识问答
  • 认知诗学解析：基于大语言模型的单意象深度分析
  • 统计分析：多维度数据聚合统计与交叉分析
  • 数据导出：支持 CSV、JSON、分析报告三种格式
  • 系统管理：系统状态监控、缓存管理、日志查看

1.3 运行环境

项目要求
操作系统Windows 10/11、Linux、macOS
Python 版本Python 3.10 及以上
Web 浏览器Chrome 90+、Edge 90+、Firefox 90+
大模型服务兼容 OpenAI API 格式的 LLM 服务（本地或远程）
磁盘空间不低于 500 MB（含向量库和日志）

二、安装与部署

2.1 环境要求

本系统依赖以下 Python 第三方库：

依赖库版本要求用途
Flask≥ 3.0.0Web 框架
Requests≥ 2.28.0HTTP 请求
ChromaDB≥ 0.4.0向量数据库
openpyxl≥ 3.1.0Excel 导出（可选）
fpdf2最新版PDF 生成（可选）

2.2 安装步骤

步骤一：确保已安装 Python 3.10 及以上版本。

python --version

步骤二：在项目目录下安装依赖。

cd 唐诗意象智能分析系统
pip install -r requirements.txt

步骤三：（可选）如果缺少 ChromaDB 等库，可单独安装。

pip install flask requests chromadb openpyxl fpdf2

2.3 启动方式

2.3.1 启动 Web 服务

在项目根目录下执行：

python app.py

看到以下输出即表示启动成功：

Running on http://127.0.0.1:5000
Running on http://192.168.1.xxx:5000

在浏览器中访问 http://127.0.0.1:5000 即可进入系统首页。

2.3.2 构建向量库（首次使用）

如果希望使用 AI 问答功能，需要先构建向量数据库：

python build_rag.py

该脚本会读取 json_dataa/ 目录下的诗歌 JSON 文件，生成向量索引并保存到 rag_db/ 目录。

2.3.3 CLI 命令行工具

系统还提供了命令行管理工具：

python cli.py status        # 查看系统状态
python cli.py scan          # 扫描数据目录
python cli.py test          # 运行单元测试
python cli.py export -f csv # 导出数据
python cli.py check-rag     # 检查向量库状态
python cli.py clear-cache   # 清理缓存

三、功能介绍

3.1 数据图谱

数据图谱是系统的主页面，提供意象数据的可视化展示和交互式浏览功能。

3.1.1 统计概览

页面顶部以统计条形式展示三项核心指标：净解析诗歌数量、提取真意象条目总数、分类维度信息。同时还提供"回收站"入口按钮，方便恢复已删除的意象条目。

【图3-1】数据图谱首页 — 统计概览与图表区域

3.1.2 核心意象 Top50

以柱状图展示出现频率最高的 50 个意象文本。图表支持鼠标悬停查看详情、拖拽缩放（底部滑块）、点击柱条自动筛选对应意象在数据表中的记录。右上角提供"放大视图"按钮，支持全屏查看。

3.1.3 意象分类域分布

按意象的大类归属展示分布情况（如自然意象-天文意象、自然意象-动物意象、人文意象-文化意象等）。同样支持点击筛选联动功能。

3.1.4 数据筛选与溯源表

数据表位于页面下方，提供以下筛选方式：

  • 文本搜索：按意象文本关键字搜索（如输入"月"可检索所有含"月"的意象）
  • 分类域筛选：按意象大类归属过滤
  • 体裁筛选：按诗歌体裁/标签过滤
  • 情感倾向筛选：按情感类别过滤
【图3-2】数据筛选与溯源数据表

数据表每行显示：诗歌编号、意象文本、所属诗歌、体裁/标签、大类归属、情感倾向、操作按钮。支持分页浏览，每页显示 25 条记录。每条记录可进行以下操作：

  • 详情：弹出模态框显示该意象的完整标注信息
  • 解析：打开认知诗学解析抽屉
  • 删除：将条目移入回收站

3.2 AI 智能问答

AI 问答功能基于 RAG（检索增强生成）架构，结合向量检索与大语言模型生成能力，实现基于诗歌原文资料的智能问答。

3.2.1 提问方式

在输入框中输入关于唐诗的问题，例如：

  • "杜甫写过哪些关于秋天的诗？"
  • "李白的《静夜思》中用了哪些意象？"
  • "唐诗中'月'的意象有哪些情感含义？"
【图3-3】AI 问答界面 — 输入问题与检索结果

3.2.2 检索与回答

点击"检索并生成回答"按钮后，系统会：

  1. 自动识别问题中涉及的诗人（如有），进行针对性过滤
  2. 在向量库中执行语义检索，找到最相关的诗歌原文
  3. 左侧面板展示检索到的诗歌卡片（含相似度评分）
  4. 右侧区域以流式方式逐字输出大模型生成的回答

诗歌卡片点击可展开查看全文。

【图3-4】AI 问答 — 回答结果与检索诗歌展示

3.2.3 多轮追问

在回答生成完成后，可在底部输入框继续追问，系统会保留对话历史进行上下文理解，实现多轮交互式问答。

3.3 认知诗学意象解析

本功能面向专业研究场景，对单个意象进行深度认知诗学分析。在数据表的任意意象条目上点击"解析"按钮，即可从右侧展开解析抽屉。

3.3.1 解析界面

解析抽屉包含以下区域：

  • 标题区域：显示意象文本、所属诗歌名称、所在诗句
  • 标签区：展示该意象的结构化标注（感知通道、分类、表现功能、认知强度、情感信息等）
  • 对话区：显示 AI 解析内容和交互历史
  • 输入区：支持多轮追问
【图3-5】认知诗学意象解析 — 右侧抽屉界面

3.3.2 解析维度

系统从以下五个维度对意象进行深度解析：

  1. 感知层：分析意象触发的感官体验和通感效果
  2. 文化层：追溯意象的典故来源和文化符号意义
  3. 情感层：基于情感极性标注分析意象的情感功能
  4. 结构层：分析意象在诗歌中的结构作用
  5. 跨诗比较：比较同一意象在不同诗歌中的异同

解析结果支持 Markdown 渲染，以结构化方式呈现分析内容。抽屉宽度支持拖拽调整，方便对照查看。

五、维护说明

5.1 日志查看

系统日志默认保存到项目根目录的 logs/app.log 文件中。日志采用滚动写入方式，单个日志文件最大 10 MB，保留最近 5 个备份文件。日志级别默认为 INFO，可通过 config.py 中的 LOG_LEVEL 配置项调整。

日志格式：

2026-05-10 14:24:50,052 [INFO] app: 启动系统 (http://0.0.0.0:5000)

控制台也会同步输出日志信息，方便实时监控。

5.2 缓存清理

系统采用双层缓存策略：

  • 内存缓存：存储最近访问的数据，读写速度快，程序重启后清空
  • 文件缓存：持久化到磁盘 cache/ 目录，跨会话保留

两种缓存均支持 TTL（生存时间）自动过期，默认 5 分钟。可在管理后台点击"清理缓存"按钮手动清理，或使用命令行：

python cli.py clear-cache

5.3 向量库管理

向量数据库保存在 rag_db/ 目录。如需重新构建，运行：

python build_rag.py

该操作会删除旧库并重新读取 json_dataa/ 目录下的所有 JSON 文件生成向量索引。

检查向量库状态：

python cli.py check-rag

5.4 常见问题

Q1：启动后浏览器访问不到页面？

请确保已在终端中运行 python app.py 且保持终端窗口不关闭。在浏览器地址栏输入 http://127.0.0.1:5000 而非 0.0.0.0:5000。

Q2：AI 问答没有返回结果？

请检查：① 是否已运行 python build_rag.py 构建向量库；② 大模型 API 服务是否正常运行（默认连接 http://127.0.0.1:44787）；③ 可在 config.py 中修改 API 地址和模型名。

Q3：如何修改服务端口？

编辑 config.py 中的 FLASK_PORT 配置项，默认 5000，修改后重启服务生效。

Q4：数据源文件放哪里？

JSON 格式的诗歌标注数据放在 poem_json/ 目录，系统启动时自动加载。RAG 构建所需的 JSON 数据放在 json_dataa/ 目录。

Q5：如何在其他电脑上使用？

在同一局域网内，其他电脑可通过 http://192.168.x.x:5000（服务器的局域网 IP）访问。如需外网访问，请配置反向代理（如 Nginx）。

Q6：控制台输出乱码怎么办？

在 Windows 下执行命令时加 PYTHONIOENCODING=utf-8 前缀，如：

PYTHONIOENCODING=utf-8 python app.py
