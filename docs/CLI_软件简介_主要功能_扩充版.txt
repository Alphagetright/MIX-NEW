本系统 V1.0.0 版本包含 6 大功能分类、17 个 CLI 命令、1 个 Agent 智能
对话模式，功能概览如下：

（一）交互式 Shell 环境
系统提供基于 prompt_toolkit 的交互式 REPL（Read-Eval-Print Loop），
支持斜杠命令自动补全、命令历史回溯、Rich 终端渲染（Panel 面板 / Table
表格 / 彩色状态标识）。并支持非交互式单命令模式，可直接通过命令行参
数执行指定操作后自动退出，适用于脚本编排和自动化任务。

（二）数据运维命令集（17 个命令）
/status：一键查看系统综合状态，涵盖 7 项目录状态、缓存状态和系统资源
使用情况。/scan：扫描数据目录，统计文件数、总大小、JSON 合法性校验，
支持格式过滤与递归开关。/export：多格式数据导出（CSV/JSON/XML/TXT/
HTML），支持字段筛选与行数限制，UTF-8-sig 编码确保 Excel 中文兼容。
/search-rag：基于 ChromaDB 向量数据库的语义级检索，2560 维向量相似度
匹配，检索范围覆盖 15081 首诗歌。/health：自动执行 8 项健康检查（目录
权限、文件合法性、数据库可用性、Python 依赖、系统资源等），输出通过率
和修复建议。/monitor-snap：实时采集 CPU、内存、磁盘、网络等系统指标快
照，支持历史记录回溯与多快照对比。/report：自动生成多维度运维报告，支
持文本/JSON/HTML 三种输出格式。/check-rag / /build-rag：向量数据库状
态检查与重建。/clear-cache / /list-exports：缓存清理与导出文件管理。
/clean-logs / /config-info / /model：日志清理、配置查看、模型切换。
/backup：全量数据备份，保留目录结构与文件元信息。/test：模块导入与功
能定性测试。/resume / /session：会话历史查看与恢复，支持跨会话连续对话。

（三）Agent 智能对话模式
区别于传统 CLI 的精确命令模式，用户可直接以自然语言描述意图（无需斜杠
前缀），系统通过大语言模型自动理解意图、编排工具调用链并执行。Agent 模
式挂载 15 个标准化工具，涵盖数据查询、系统运维、专家分析等类别。核心
亮点为"专家协作分析引擎"——当用户发出诗歌分析请求时，系统自动启动 5 位
AI 专家并行工作：意象识别专家、格律修辞专家、情感分析专家、典故溯源专
家、历史考证专家，各专家独立生成分析报告后由协作商榷层综合形成最终结论，
模拟学术团队的多维度研讨模式。

（四）向量语义检索引擎
集成 ChromaDB 向量数据库与 text-embedding-qwen3-embedding-4b 嵌入模型
（2560 维），对诗歌原文构建语义索引。不同于传统关键词匹配，语义检索可
理解查询意图的深层含义，有效解决古典诗歌中同义词、近义词、隐喻表达的检
索难题。

（五）会话持久化与恢复
每次交互自动生成会话记录（JSON 格式持久化存储），记录用户输入、Agent 回
复、工具调用历史及时间戳。支持通过 /resume 查看全部历史会话列表，指定会
话 ID 恢复或 /resume last 恢复最近会话，实现跨会话的连续工作上下文。

（六）分层架构设计
系统采用六层架构：入口层（模式路由）→ 渲染层（Rich UI）→ 逻辑层（命令
调度）→ 服务层（LLM/Agent/RAG）→ 会话层（持久化）→ 数据层（配置/模型
定义）。各层通过明确接口通信，模块间低耦合高内聚，支持灵活扩展。
