第二节 项目介绍
1.2.1 项目简介
《唐诗意象智能分析系统》项目是顺应数字人文研究与 AI 智能技术深度融合的趋势而开发的。随着古典文学研究向数字化、智能化转型，NLP（自然语言处理）与知识检索技术在唐诗分析、意象统计等场景的需求持续增长；但当前市场中，多数文学研究工具存在数据分散（诗歌文本与结构化标注分离）、智能交互与学术研究场景脱节、数据成果共享效率低等问题，难以满足文学研究者的综合需求，为此，我们启动了本项目的开发。
本项目重点打造唐诗意象的"数据图谱可视化"与"AI 智能问答"两大核心能力：基于 Flask Web 框架构建系统骨架，采用 ECharts 实现意象数据的多维度图表展示，结合 ChromaDB 向量数据库与大语言模型构建 RAG 检索增强问答引擎。系统内置完整的唐诗结构化数据——313 首唐诗、12,738 条分析单元、4,896 条标注意象，覆盖自然意象、社会意象、人文意象三大分类体系；同时提供认知诗学 AI 解析功能，支持从感知层、文化层、情感层、结构层、跨诗比较五个维度对单条意象进行深度解读。系统既是文学研究者的"数据分析助手"，又是唐诗教学的"智能教辅工具"。
1.2.2 功能介绍
一、数据图谱模块
该模块是唐诗意象智能分析系统中诗歌数据可视化管理与多维度分析的核心入口，为文学研究者、教育工作者提供一站式的意象数据查看与统计分析服务，实现"可视化图表 + 数据筛选 + 溯源浏览"的一体化操作闭环。在统计概览方面，模块顶部展示净解析诗歌总数、提取意象条目总数、分类维度说明等关键指标，让用户快速掌握数据全貌。
在核心意象 Top50 图表方面，模块基于 ECharts 柱状图展示出现频次最高的 50 个意象文本，横轴为意象名称、纵轴为频次，支持数据缩放滑块与全屏放大视图，点击柱子即可自动筛选溯源数据表中对应的意象条目。在意象分类域分布图表方面，模块以柱状图展示各分类域（如自然意象-天文意象、社会意象-生产生活意象等）的意象分布情况，支持点击筛选与全屏视图。
在数据筛选方面，模块提供文本搜索框、分类域下拉筛选、体裁/标签下拉筛选、情感倾向下拉筛选，支持多条件组合过滤，并有清除筛选按钮。在溯源数据表方面，模块以分页表格展示每条意象的详细信息（诗歌编号、意象文本、所属诗歌、体裁/标签、大类归属、情感倾向），每页25条，最多渲染5000条，支持详情查看、认知诗学解析、删除等操作。
在意象详情查看方面，点击详情按钮弹出模态框，展示完整的结构化标注信息（词性、成分类型、感知通道、素材类型、内部结构、指涉来源、表现功能、文化流通性、跨文化性、认知强度、核心意象、情感极性、情感类别、情感置信度、大类编码、子类编码等25+字段）及四层摘要和完整诗句原文。在回收站功能方面，删除的意象条目移入回收站，支持单条恢复和计数更新。
二、AI 智能问答模块
该模块是基于检索增强生成（RAG）架构的唐诗知识问答系统，为用户提供"检索-生成-追问"的完整智能问答闭环。在提问方式方面，用户在输入框中输入关于唐诗的自然语言问题（如"杜甫写过哪些关于秋天的诗？"），系统自动从问题中识别诗人姓名并提取语义查询关键词。在检索与回答流程方面，系统将问题向量化后在 ChromaDB 向量库中检索 Top-5 相关诗歌，支持按作者过滤检索，将检索结果构建为上下文后调用大语言模型生成高质量回答，并通过 SSE 流式传输至前端展示。在检索结果展示方面，前端采用双栏布局：左栏展示检索到的诗歌卡片（标题、作者、相似度、原文预览、可展开全文），右栏实时展示流式回答。在多轮追问方面，系统保存对话历史结构，追问时携带完整历史记录，无需重复检索即可继续深入探讨。
三、认知诗学意象解析模块
该模块是对单条意象数据进行深度认知诗学解析的 AI 分析工具，采用"右侧抽屉"交互模式。在解析维度方面，系统从五个维度进行解析：感知层（触发的感官体验和通感效果）、文化层（典故来源和文化符号意义）、情感层（基于情感极性分析情感功能）、结构层（作为特定结构功能组在诗中的结构作用）、跨诗比较（此意象在其他诗歌中的异同）。在交互方式方面，点击溯源表中的"解析"按钮从右侧滑入抽屉，展示意象元信息与标签 chips，对话式消息列表支持 Markdown 渲染，首轮自动发送默认解析请求，支持多轮追问与流式渲染，解析结果缓存避免重复请求，抽屉可拖拽调整宽度。
四、管理后台模块
该模块是系统运维管理的核心入口，为管理人员提供系统状态监控与运维操作能力。在系统状态面板方面，展示净解析诗歌数、意象条目数、分类维度数、已知诗人数、向量数据库状态、缓存状态、导出文件数等关键指标。在可视化图表方面，展示意象分类域分布图与情感类别分布图。在操作面板方面，支持刷新数据缓存、导出 CSV/JSON/分析报告、清理全部缓存。在导出文件管理方面，列出所有导出文件（文件名、大小、修改时间），支持清空导出目录。
五、数据导出模块
该模块支持三种格式的数据导出：CSV 导出包含 24 个字段的溯源表数据（UTF-8-BOM 编码）；JSON 导出完整数据字典；统计报告导出多维度分析摘要。同时提供导出文件列表查看和清空导出目录功能。
六、CLI 命令行工具模块
该模块提供 8 个子命令的系统运维工具：status（查看系统状态）、scan（扫描数据目录）、export（导出数据至 CSV/JSON/report）、clear-cache（清理缓存）、list-exports（列出导出文件）、check-rag（检查向量库状态）、build-rag（构建向量库）、test（运行单元测试）。

3.2.2 后台 API 接口说明
系统提供 14 个 RESTful API 接口，分为前端 API 和管理后台 API 两类。
前端 API：
● GET / ：返回系统首页 HTML。
● GET /api/data ：返回全部数据（含 total_poems、total_images、traceback 数组）。
● GET /api/stats ：返回多维度统计分析概览（含 summary、cross_analysis）。
● POST /api/export ：导出数据，参数 {format: "csv"|"json"}。
● POST /api/ask ：RAG 智能问答，参数 {question, history[]}，返回 SSE 流（type: poems/chunk/error）。
● POST /api/analyze ：认知诗学解析，参数 {item, history[], followup}，返回 SSE 流（type: chunk/text/error）。
● GET /api/cache/status ：返回缓存状态（memory_entries、file_entries、data_cache_active）。
管理后台 API：
● GET /admin/ ：返回管理后台首页 HTML。
● GET /admin/api/status ：返回系统状态总览（含 data、rag_database、cache、config、exports 等）。
● GET /admin/api/stats ：返回详细统计数据（含 category_stats、emotion_stats、author_stats 等）。
● GET /admin/api/exports ：列出导出文件列表。
● POST /admin/api/exports ：创建导出，参数 {format: "csv"|"json"|"report"}。
● POST /admin/api/exports/clear ：清空导出目录。
● POST /admin/api/cache/clear ：清理缓存，参数 {type: "memory"|"file"|"all"}。
● POST /admin/api/refresh ：刷新数据缓存。

第三节 核心数据结构
3.3.1 诗歌数据模型（models.py）
系统使用 Python dataclass 定义了 7 个数据模型类：
● PoemLine（诗行）：诗行编号、原文、注释。
● AnalysisUnit（分析单元）：25 个字段——单元编号、诗行编号、文本、行内位置、词性、成分类型、是否意象、大类编码、子类编码、感知通道、素材类型、内部结构、指涉来源、表现功能、文化流通性、跨文化性、认知强度、核心意象、结构功能组、情感极性、情感类别、情感置信度。
● EmotionTrajectory（情感轨迹）：诗行编号、诗行原文、平均情感极性、主导情感、情感波动值。
● ImageRelation（意象关系）：关系编号、来源单元编号、来源文本、目标单元编号、目标文本、关系类型、关系强度。
● Poem（诗歌）：诗歌编号、标题、作者、朝代、分类标签、体裁、原文、诗行列表、分析单元列表、情感轨迹列表、意象关系列表。
● TracebackItem（溯源条目）：23 个平面化字段，用于前端表格展示。
● AnalysisResult（解析结果）：item、messages、done、is_streaming 状态字段。
